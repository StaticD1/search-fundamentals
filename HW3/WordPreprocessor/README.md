## Проект: Токенизация, лемматизация и булев поиск

Данный проект реализует два последовательных этапа обработки текстовых документов (HTML-страниц в формате .txt):

Токенизация и лемматизация – извлечение чистого текста из HTML, выделение уникальных токенов (слов), приведение их к
нормальной форме (лемме для русского языка, стеммингу для английского), группировка токенов по леммам для каждого
документа.

Построение инвертированного индекса по леммам и булев поиск с поддержкой операторов AND, OR, NOT и скобок. Запрос
вводится пользователем.

Проект написан на Java с использованием Spring Boot

### Особенности

* Агрегация данных: Результаты обработки всех страниц собираются в два общих файла.
* Очистка HTML: Использование библиотеки Jsoup для корректного удаления тегов, скриптов и стилей.
* Токенизация: Разбиение текста на отдельные слова с удалением дубликатов.
* Фильтрация:
    * Удаление стоп-слов (союзы, предлоги) на базе стандартного списка Apache Lucene.
    * Удаление "мусора" (слова с цифрами, обрывки разметки) через регулярные выражения.
* Лемматизация: Приведение слов к основе с использованием алгоритма Стеммера Портера (Snowball) для русского языка.

## Установка и запуск

### Предварительные требования

* JDK 17 или выше
* Gradle (или можно использовать обёртку ./gradlew)

### Шаг 1: Сборка

Склонируйте проект и выполните команду:

```bash
./gradlew clean build
```

### Шаг 2: Подготовка данных

Создайте папку data/input в корне проекта.
Поместите туда файлы .txt, содержащие HTML-разметку.

### Шаг 3: Запуск

```bash
./gradlew bootRun
```

### После запуска приложение:

1. Создаст файл с инвертированным индексом по всем файлам *_lemmas.txt из папки output и .
2. Выведет количество загруженных лемм и документов.
3. Перейдет в интерактивный режим ожидания запросов.

### Использование булева поиска
Введите запрос в консоли и нажмите Enter. Для выхода введите exit.

### Поддерживаемый синтаксис
* Термы – отдельные слова (могут содержать дефис). Регистр не важен.
* Операторы (регистронезависимые):
  * AND – логическое И
  * OR – логическое ИЛИ
  * NOT – логическое НЕ (унарный оператор)
* Круглые скобки – для изменения приоритета.

Приоритет операторов (от высокого к низкому):
NOT > AND > OR

### Примеры запросов
```text
Клеопатра AND Цезарь
(Клеопатра AND Цезарь) OR (Антоний AND Цицерон) OR Помпей
NOT Антоний
Помпей AND NOT Цезарь
(Антоний OR Цезарь) AND (Клеопатра OR Помпей)
```
### Как работает нормализация термов
* Для русских слов выполняется лемматизация (приведение к начальной форме) с помощью библиотеки AOT. Например, «Клеопатры», «Клеопатре» → «клеопатра».
* Для английских слов применяется стемминг Портера. Например, «running», «ran» → «run».

Благодаря этому запросы можно вводить в любой словоформе, и они будут корректно сопоставлены с леммами в индексе.

### Результат поиска
На экран выводится количество найденных документов и список их идентификаторов (имена исходных файлов без суффикса _lemmas.txt). Если документов много, они выводятся по одному в строке.

### Примечания
* Файлы tokens.txt и lemmas.txt в папке output игнорируются при построении индекса.
* Если запрос синтаксически некорректен (например, непарные скобки), будет выведено сообщение об ошибке.
* Термы, отсутствующие в индексе, считаются соответствующими пустому множеству документов.

### Возможные улучшения

* Расширенный вывод с информацией о документах (заголовки, URL).
* Кэширование результатов для повторяющихся запросов.
