## Проект: Токенизация, лемматизация, tf-idf и булев поиск
Данный проект представляет собой комплексное решение для обработки коллекции HTML-документов (в формате .txt) с целью:

Токенизации и лемматизации – извлечение чистого текста из HTML, выделение уникальных токенов, приведение их к нормальной форме (леммы для русского языка, стемминг для английского), группировка по леммам для каждого документа.

Расчёта tf-idf – вычисление term frequency (tf) и inverse document frequency (idf) для каждого токена и каждой леммы, а также tf-idf; результаты сохраняются в отдельных файлах для каждого документа.

Построения инвертированного индекса – по леммам для быстрого поиска.

Булева поиска – интерактивный поиск по индексу с поддержкой операторов AND, OR, NOT и круглых скобок.

Проект реализован на Java с использованием Spring Boot

### Особенности

* Агрегация данных: Результаты обработки всех страниц собираются в два общих файла.
* Очистка HTML: Использование библиотеки Jsoup для корректного удаления тегов, скриптов и стилей.
* Токенизация: Разбиение текста на отдельные слова с удалением дубликатов.
* Фильтрация:
    * Удаление стоп-слов (союзы, предлоги) на базе стандартного списка Apache Lucene.
    * Удаление "мусора" (слова с цифрами, обрывки разметки) через регулярные выражения.
* Лемматизация: Приведение слов к основе с использованием алгоритма Стеммера Портера (Snowball) для русского языка.

### Конфигурация

Все настройки задаются в файле application.properties:
```properties
# Относительные пути к директориям с данными
app.input-dir=dir
app.output-dir=dir
# Включать ли расчёт tf-idf при запуске (true/false)
tfidf.enabled=true
```

* Если tfidf.enabled=true, то перед построением индекса и запуском поиска будет выполнен полный пересчёт tf-idf для всех документов из папки input. Результаты сохраняются в подпапках output/tfidf_tokens_per_doc и output/tfidf_lemmas_per_doc.

* Если tfidf.enabled=false, этап расчёта пропускается, и приложение сразу строит индекс по существующим файлам лемм (*_lemmas.txt) в папке output.

## Установка и запуск

### Предварительные требования

* JDK 17 или выше
* Gradle (или можно использовать обёртку ./gradlew)

### Шаг 1: Сборка

Склонируйте проект и выполните команду:

```bash
./gradlew clean build
```

### Шаг 2: Подготовка данных

Создайте папку data/input в корне проекта.
Поместите туда файлы .txt, содержащие HTML-разметку.

### Шаг 3: Запуск

```bash
./gradlew bootRun
```

### После запуска приложение:

1. Создаст файл с инвертированным индексом по всем файлам *_lemmas.txt из папки output и .
2. Выведет количество загруженных лемм и документов.
3. Перейдет в интерактивный режим ожидания запросов.

### Расчёт tf-idf
#### Алгоритм
1. Для каждого HTML-файла из папки input:
   * Извлекается текст с помощью Jsoup. 
   * Выполняется токенизация (разбиение на слова, приведение к нижнему регистру) с использованием регулярного выражения \p{L}+ (все буквенные символы Unicode).
   * Подсчитывается частота каждого токена (сколько раз встретился в документе) и общее количество токенов в документе.
   * Для каждого токена определяется лемма с помощью LemmatizerService (русские слова лемматизируются через AOT, английские – стеммингом Портера). Накопляется частота каждой леммы.
2. После обработки всех документов:
   * Для каждого уникального токена и каждой уникальной леммы вычисляется document frequency (df) – количество документов, в которых встречается данный токен/лемма.
   * Вычисляется idf = ln(N / df), где N – общее число документов.
3. Для каждого документа:
   * Для каждого токена: tf = частота токена / общее число токенов в документе, tf-idf = tf × idf(токена). Результат записывается в файл tfidf_tokens_per_doc/<docId>_tfidf_tokens.txt.
   * Для каждой леммы: tf = частота леммы / общее число токенов в документе, tf-idf = tf × idf(леммы). Результат записывается в файл tfidf_lemmas_per_doc/<docId>_tfidf_lemmas.txt.
#### Формат выходных данных
Каждая строка содержит 3 значения, разделённым пробелом:
```text
<термин/лемма> <idf> <tf-idf>
```
Числа выводятся с шестью знаками после запятой (можно изменить формат в коде).

Пример для токенов (doc1_tfidf_tokens.txt):
```text
клеопатра 0.693147 0.023456
цезарь 1.098612 0.045678
антоний 0.356675 0.012345
...
```
Аналогично для лемм.

### Использование булева поиска
Введите запрос в консоли и нажмите Enter. Для выхода введите exit.

### Поддерживаемый синтаксис
* Термы – отдельные слова (могут содержать дефис). Регистр не важен.
* Операторы (регистронезависимые):
  * AND – логическое И
  * OR – логическое ИЛИ
  * NOT – логическое НЕ (унарный оператор)
* Круглые скобки – для изменения приоритета.

Приоритет операторов (от высокого к низкому):
NOT > AND > OR

### Примеры запросов
```text
Клеопатра AND Цезарь
(Клеопатра AND Цезарь) OR (Антоний AND Цицерон) OR Помпей
NOT Антоний
Помпей AND NOT Цезарь
(Антоний OR Цезарь) AND (Клеопатра OR Помпей)
```
### Как работает нормализация термов
* Для русских слов выполняется лемматизация (приведение к начальной форме) с помощью библиотеки AOT. Например, «Клеопатры», «Клеопатре» → «клеопатра».
* Для английских слов применяется стемминг Портера. Например, «running», «ran» → «run».

Благодаря этому запросы можно вводить в любой словоформе, и они будут корректно сопоставлены с леммами в индексе.

### Результат поиска
На экран выводится количество найденных документов и список их идентификаторов (имена исходных файлов без суффикса _lemmas.txt). Если документов много, они выводятся по одному в строке.

### Примечания
* Файлы tokens.txt и lemmas.txt в папке output игнорируются при построении индекса.
* Если запрос синтаксически некорректен (например, непарные скобки), будет выведено сообщение об ошибке.
* Термы, отсутствующие в индексе, считаются соответствующими пустому множеству документов.

### Возможные улучшения
* Поддержка фразовых запросов (в кавычках).
* Расширенный вывод с информацией о документах (заголовки, URL).
* Кэширование результатов для повторяющихся запросов.
